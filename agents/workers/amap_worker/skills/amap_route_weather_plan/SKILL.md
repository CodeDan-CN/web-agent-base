# amap_route_weather_plan

## 用途

根据起点和终点地址，完成地址解析、驾车路线查询、目的地天气查询和出行建议生成。

## 适用场景

- 用户提供起点和终点自然语言地址。
- 用户需要路线、天气和出行建议的组合结果。
- 用户不想分别提供经纬度和 adcode。

## 不适用场景

- 用户没有提供起点或终点。
- 地址过于模糊，无法解析到具体位置。

## 调用注意事项

- 必须提供 `origin_address` 和 `destination_address`。
- `city` 可选，但建议提供同城出行城市。
- 本 Skill 是固定链路，不允许模型动态生成执行步骤。
