# 第三阶段测试用例

## 概述

本文档定义第三阶段「Worker Agent 闭环 + 事件内对话预加载」测试用例。

第三阶段测试目标是验证：

- main agent 可以扫描和调用真实 worker agent。
- worker agent 可以加载自己的 Agent 资产和 SkillCatalog。
- main agent 不直接持有高德领域 Skill。
- 高德相关请求由 main agent 调用 `amap_worker` 处理。
- `amap_worker` 内部可以调用 `api / chain / code` Skill。
- worker 执行结果可以回到 main agent，并由 main agent 最终回答用户。
- main / worker 共享同一个 `event_id`。
- 事件结束后可以通过简易 Hook 生成 main / worker 各自的事件 summary。
- 下一次 Loop 前可以执行 `EventContextPreloader`。
- Prompt 中只注入命中事件内与当前 agent 相关的一轮或多轮对话，不注入 `event_id`、事件标题、summary、worker / Skill 执行摘要。
- 缺参、多轮补充、失败和 AG-UI 中间步骤都可运行。

## 一、预置 Worker

第三阶段预置一个真实 worker：

| worker_id | 用途 |
| --- | --- |
| `amap_worker` | 高德 Web 服务领域 worker，处理地址、路线、天气、行政区域、IP 定位、坐标转换 |

### 1.1 amap_worker Skill

| skill_id | 类型 | 用途 |
| --- | --- | --- |
| `amap_geocode` | `api` | 地址转经纬度、adcode、citycode |
| `amap_regeocode` | `api` | 经纬度转结构化地址 |
| `amap_weather` | `api` | 根据 adcode 查询实时天气或天气预报 |
| `amap_district_search` | `api` | 行政区域查询 |
| `amap_ip_location` | `api` | IPv4 地址定位 |
| `amap_coordinate_convert` | `api` | 坐标系转换 |
| `amap_route_driving` | `api` | 驾车路径规划 |
| `amap_route_walking` | `api` | 步行路径规划 |
| `amap_route_transit` | `api` | 公交路径规划 |
| `amap_route_distance` | `api` | 距离测量 |
| `travel_briefing_formatter` | `code` | 路线和天气结果格式化 |
| `amap_route_weather_plan` | `chain` | 地址解析 → 路线 → 目的地天气 → 出行建议 |

## 二、测试原则

第三阶段测试分六层：

1. Worker 资产加载测试。
2. Worker Skill 迁移测试。
3. main → worker → skill 全链路测试。
4. 缺参与多轮补充测试。
5. 事件 summary 与 Loop 前事件内对话预加载测试。
6. AG-UI 中间步骤测试。

测试必须保持：

- 单轮简单问题使用新 `session_id`。
- 多轮测试每组使用独立新 `session_id`，组内复用同一个 `session_id`。
- 事件相关测试每组使用独立新 `user_id + session_id`，避免历史事件互相污染。
- 高德领域请求只要可以识别 worker，就应进入 `amap_worker`。
- main agent 不直接调用高德 Skill。
- 用户消息使用正常表达，不靠测试暗号触发链路。
- 用户侧输出不能暴露 worker 内部 Prompt、Adapter、schema、API Key、异常堆栈。
- main 直接 `ask_user → awaiting_user` 只用于无法判断应交给哪个 worker 的场景。
- 用户补充信息后，必须能回到 `call_agent(amap_worker)` 链路。
- 事件相关性判断只允许在当前会话候选事件中选择一个命中事件。
- Prompt 注入只允许注入命中事件内的一轮或多轮对话。

## 三、统一请求格式

所有 Agent 链路测试通过 `/agent/run` 或 `/agent/run/stream` 触发。

```json
{
  "user_id": "worker_test_user",
  "agent_id": "main",
  "session_id": null,
  "request_id": "case_worker_001",
  "message": "我从杭州市西湖区文三路开车去杭州市上城区湖滨银泰，帮我看路线和目的地天气。",
  "metadata": {}
}
```

事件显式延续测试可以在 `metadata` 中传入：

```json
{
  "event_id": "existing_event_id"
}
```

## 四、轨迹采集

每个用例需要采集：

```text
request
main_agent_run
main_agent_run_step[]
worker_agent_run
worker_agent_run_step[]
agent_event
agent_event_run[]
agent_event_context[]
main_tool_call[]
worker_tool_call[]
final_response
session_state
ag_ui_events[]
```

检查重点：

```text
main tool_call.tool_name = amap_worker
worker tool_call.tool_name = 具体 skill_id
main agent_run_step.action_detail 包含轻量 worker handoff
main tool_call.input_payload 或 worker run 输入摘要包含完整 Worker Task Package
agent_event_run 能关联 main run 和 worker run
agent_event_context 分别存在 main 和 amap_worker 的 summary
main final_state = completed 或 awaiting_user
worker final_state = completed 或 awaiting_user
```

事件内对话预加载额外检查：

```text
session_context.event_context.should_inject
session_context.event_context.event_dialogues
Prompt 输入中不存在 event_id / title / summary / worker 执行摘要
Prompt 输入中只存在命中事件内的一轮或多轮对话
```

AG-UI 流式用例至少检查：

```text
RUN_STARTED
STATE_SNAPSHOT
STEP_STARTED
STATE_DELTA
TOOL_CALL_START
TOOL_CALL_ARGS
TOOL_CALL_END
TEXT_MESSAGE_START
TEXT_MESSAGE_CONTENT
TEXT_MESSAGE_END
RUN_FINISHED
```

## 五、资产和迁移测试

### 5.1 Worker 资产加载

检查点：

- `agents/workers/amap_worker/` 存在。
- 必要文件齐全：

```text
SOUL.md
Agent.md
Instruction.md
tools.md
output.md
harness.md
prompts/fragments.md
prompts/examples.md
```

- `amap_worker` 可以构建 AgentDefinition。
- `amap_worker.kind = worker`。
- `amap_worker.skill_registry` 可以列出自己的 SkillCatalog。

### 5.2 main agent 不直接持有高德 Skill

检查点：

- `agents/main/skills/` 只保留通用 Skill。
- `content_extract` 仍可由 main agent 使用。
- 以下 Skill 不应出现在 main SkillCatalog：

```text
amap_geocode
amap_regeocode
amap_weather
amap_route_driving
amap_route_weather_plan
travel_briefing_formatter
```

### 5.3 amap_worker 持有高德 Skill

检查点：

- `amap_worker` SkillCatalog 包含第一批高德 Skill。
- 每个 Skill 都有 `SKILL.md` 和 `schema.json`。
- 每个 `schema.json` 的 `skill_id` 与目录名一致。
- 每个 Skill 的输入和输出 schema 可通过校验。

### 5.4 Worker Task Package 结构

用户输入：

```text
我从杭州市西湖区文三路开车去杭州市上城区湖滨银泰，帮我看路线和目的地天气。
```

检查点：

Main LoopDecider 输出：

- main 的 `call_agent` action_detail 包含 `worker_id = amap_worker`。
- action_detail 包含 `task`。
- action_detail 包含 `handoff_context`，内容是 main 已理解到的自然语言关键信息。
- action_detail 包含 `handoff.reason`。
- action_detail 不包含 `context_package`。
- action_detail 不包含 `output_contract`。

Runtime 补齐后的 Worker Task Package：

- Worker Task Package 包含 `context_package.event_id`。
- Worker Task Package 包含 `context_package.user_message`。
- Worker Task Package 包含 `context_package.conversation_summary`。
- Worker Task Package 包含 `context_package.event_history`。
- Worker Task Package 包含 `context_package.recent_turns`。
- Worker Task Package 包含 `context_package.retrieved_context`，第三阶段允许为空数组。
- Worker Task Package 包含 `context_package.long_term_memory`，第三阶段允许为空数组。
- Worker Task Package 包含 `context_package.memory_palace_refs`，第三阶段允许为空数组。
- Worker Task Package 包含 `output_contract.final_answer_owner = main_agent`。

第三阶段不要求：

- 长期记忆召回。
- 记忆宫殿召回。
- 跨会话事件检索。
- 全量 LLM 对话压缩摘要体系。

## 六、单轮 Worker 测试

### 6.1 地址解析

用户输入：

```text
帮我解析一下地址：杭州市西湖区文三路，城市是杭州。
```

期望链路：

```text
main.new_request
  → call_agent(amap_worker)
  → worker.call_skill(amap_geocode)
  → worker.answer_user
  → main.answer_user
  → completed
```

检查点：

- main `tool_call.tool_name = amap_worker`。
- worker `tool_call.tool_name = amap_geocode`。
- worker 输出包含 `location` 和 `adcode`。
- main 最终回答自然说明解析结果。

### 6.2 实时天气

用户输入：

```text
查询 adcode 330106 的实时天气。
```

期望链路：

```text
main.call_agent(amap_worker)
  → worker.call_skill(amap_weather)
  → completed
```

检查点：

- main 不直接调用 `amap_weather`。
- worker 调用 `amap_weather`。
- 输出包含天气、温度、风向、湿度。

### 6.3 驾车路径规划

用户输入：

```text
帮我查一下从 120.123456,30.123456 到 120.654321,30.654321 的驾车路线。
```

期望链路：

```text
main.call_agent(amap_worker)
  → worker.call_skill(amap_route_driving)
  → completed
```

检查点：

- worker 使用 `amap_route_driving`。
- 输出包含距离、预计时间和路线摘要。

### 6.4 地址到路线天气建议

用户输入：

```text
我从杭州市西湖区文三路开车去杭州市上城区湖滨银泰，帮我看路线和目的地天气。
```

期望链路：

```text
main.call_agent(amap_worker)
  → worker.call_skill(amap_route_weather_plan)
  → completed
```

检查点：

- worker 使用 chain skill `amap_route_weather_plan`。
- chain 内部包含地址解析、路线、天气和格式化结果。
- worker 返回结构化领域结果。
- main 最终回答面向用户，不暴露内部链路名。

## 七、缺参与多轮测试

### 7.1 缺少出发地

用户输入：

```text
帮我查一下去杭州市上城区湖滨银泰的路线和天气。
```

期望链路：

```text
main.call_agent(amap_worker)
  → worker.missing_params
  → main.harness
  → main.ask_user
  → awaiting_user
```

检查点：

- main 识别出高德领域后，应调用 `amap_worker` 获取领域缺参判断。
- 追问聚焦缺少出发地。
- 不使用当前位置、默认地点或模型猜测地点补齐参数。
- worker 返回 `missing_params`。
- main 根据 worker 缺参结果追问用户。
- 当前事件状态可进入 `awaiting_user`。

### 7.2 多轮补充完成

轮次 1：

```text
帮我规划一下明天开车出行，顺便看天气。
```

期望链路：

```text
main.call_agent(amap_worker)
  → worker.missing_params
  → main.ask_user
  → awaiting_user
```

轮次 2：

```text
我从杭州市西湖区文三路去杭州市上城区湖滨银泰。
```

期望链路：

```text
awaiting_user
  → main.call_agent(amap_worker)
  → worker.call_skill(amap_route_weather_plan)
  → completed
```

检查点：

- 第二轮复用同一个 `session_id`。
- 第二轮复用同一个 `event_id`。
- Runtime 可以承接上一轮待补充意图。
- 第二轮进入 `amap_worker`。
- worker 进入 `amap_route_weather_plan`。
- 事件结束后可生成 main / worker 的 `agent_event_context.summary`。

### 7.3 地址无法解析

用户输入：

```text
我从一个不存在的火星基地开车去杭州市上城区湖滨银泰，帮我看路线和天气。
```

允许链路：

```text
main.call_agent(amap_worker) → worker.missing_params → main.ask_user → awaiting_user
```

或：

```text
main.call_agent(amap_worker) → worker.call_skill(amap_route_weather_plan) → failed
```

检查点：

- 用户侧自然说明地址无法确认。
- 不暴露高德原始错误堆栈。
- 不编造路线结果。

## 八、事件 Summary 与 Loop 前事件内对话预加载测试

### 8.1 事件结束后生成 main / worker summary

用户输入：

```text
我从杭州市西湖区文三路开车去杭州市上城区湖滨银泰，帮我看路线和目的地天气。
```

期望链路：

```text
main.call_agent(amap_worker)
  → worker.call_skill(amap_route_weather_plan)
  → main.answer_user
  → completed
  → EventSummaryRequested
```

检查点：

- `agent_event.status = completed`。
- `agent_event_run` 至少包含 main run 和 `amap_worker` run。
- `agent_event_context(event_id, agent_id = main)` 存在。
- `agent_event_context(event_id, agent_id = amap_worker)` 存在。
- summary 可以是 direct 或 llm 模式。
- summary 生成失败不能影响用户最终回答。

### 8.2 显式 event_id 直接注入事件内对话

准备数据：

- 已存在一个 `agent_event`。
- 该事件下已有当前 agent 的一轮或多轮 `agent_run`。
- 请求 `metadata.event_id` 指向该事件。

请求：

```json
{
  "user_id": "event_test_user",
  "agent_id": "main",
  "session_id": "event_test_session",
  "request_id": "case_event_explicit",
  "message": "那附近方便停车吗？",
  "metadata": {
    "event_id": "existing_event_id"
  }
}
```

检查点：

- 不调用模型做事件相关性判断。
- `EventContextPreloader` 读取该 `event_id` 下当前 agent 的历史对话。
- `session_context.event_context.should_inject = true`。
- `session_context.event_context.event_dialogues` 包含该事件内一轮或多轮对话。
- Prompt 中不出现 `event_id`。
- Prompt 中不出现事件标题。
- Prompt 中不出现 `agent_event_context.summary`。
- Prompt 中不出现 worker / Skill 执行摘要。

### 8.3 无显式 event_id 时匹配当前会话历史事件

准备数据：

- 当前 `user_id + session_id` 下存在至少一个已完成历史事件。
- 历史事件有 `agent_event_context.summary`。
- 历史事件下有当前 agent 的一轮或多轮对话。
- 当前请求未传 `metadata.event_id`。

请求：

```text
那目的地附近有没有停车场？
```

期望链路：

```text
EventContextPreloader
  → 读取当前会话候选事件
  → 模型判断 related = true
  → 注入命中事件内对话
  → LoopDecider
```

检查点：

- 候选事件只来自当前 `user_id + session_id`。
- 模型相关性判断只接收候选事件短标题、summary、状态和更新时间。
- 如果命中，只取一个最相关事件。
- `session_context.event_context.event_dialogues` 只包含命中事件内对话。
- Prompt 中不出现 `event_id`、事件标题、summary 或执行摘要。

### 8.4 当前会话无历史事件时不注入

准备数据：

- 使用全新的 `user_id + session_id`。
- 当前会话没有历史事件。

请求：

```text
那附近方便停车吗？
```

检查点：

- 不调用模型做事件相关性判断。
- `session_context.event_context.should_inject = false`。
- `session_context.event_context.event_dialogues = []`。
- Loop 按普通新请求处理。

### 8.5 模型判断不相关时不注入

准备数据：

- 当前会话存在一个历史事件：路线和天气查询。
- 当前请求是完全无关任务。

请求：

```text
帮我把这段项目说明提炼成三点。
```

检查点：

- 模型相关性判断可以返回 `related = false`。
- 不注入历史事件对话。
- main 不因为历史事件误调用 `amap_worker`。
- 如果任务属于 main 通用 Skill，应走 `content_extract` 或直接回答。

### 8.6 worker 不重新匹配事件归属

用户输入：

```text
继续帮我看刚才那个目的地的天气。
```

前置条件：

- main 已经通过显式事件或相关事件判断确定当前 `event_id`。
- main 调用 `amap_worker`。

检查点：

- worker request metadata 继承 main 当前 `event_id`。
- worker 不执行当前会话历史事件相关性判断。
- worker 只读取 `event_id + worker_agent_id` 下自己的事件内对话。
- worker Prompt 中不出现 `event_id`、事件标题、summary 或执行摘要。

## 十一、验收要求

第三阶段测试通过条件：

- `amap_worker` 可以被 main agent 扫描和调用。
- 高德 Skill 不再出现在 main SkillCatalog。
- 高德 Skill 出现在 `amap_worker` SkillCatalog。
